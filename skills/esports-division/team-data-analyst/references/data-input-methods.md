# طرق إدخال البيانات والتحقق من القائمة

## الربط مع مدير القائمة (Roster Manager)

### آلية المزامنة
- محلل بيانات الفريق يقرأ ملف بيانات مدير القائمة (`roster_data.json`)
- يستخرج قائمة اللاعبين المسجلين (IGN, UID, الدور)
- أي تحليل يجب أن يكون للاعب موجود في هذه القائمة فقط

### التحقق من الانتماء
```
خطوات التحقق قبل أي تحليل:
1. استلام اسم اللاعب (IGN) أو UID
2. البحث في قائمة مدير القائمة
3. إن وُجد → المتابعة بالتحليل
4. إن لم يُوجد → رفض + إشعار "اللاعب ليس ضمن قائمة الفريق"
```

### البيانات المستوردة من مدير القائمة

| الحقل | الاستخدام في التحليل |
|-------|---------------------|
| IGN | معرّف اللاعب في كل التحاليل والتقارير |
| UID | ربط بيانات اللعبة باللاعب الصحيح |
| الدور_في_الفريق | تصنيف التحليل حسب الدور (IGL, Fragger, Support...) |
| الجهاز + FPS | تفسير اختلافات الأداء حسب الجهاز |
| الحالة | استبعاد اللاعبين (إصابة/غائب/معار) من التحليل الجاري |

## مصادر إدخال البيانات

### 1. إدخال يدوي
- إدخال إحصائيات المباراة مباشرة عبر السكربت
- مناسب لبيانات السكريمات والتدريبات

### 2. رفع ملفات
| نوع الملف | الامتدادات | البيانات المتاحة |
|-----------|-----------|-----------------|
| فيديو | MP4, AVI, MKV, MOV | إحصائيات مرئية، حركات، قرارت |
| صورة | PNG, JPG, JPEG | Screenshots إحصائيات اللعبة |
| جدول | CSV, XLSX, XLS | بيانات رقمية منظمة |
| مستند | PDF, DOCX | تقارير، نصوص، جداول |

### 3. روابط خارجية
| المنصة | صيغة الرابط | البيانات |
|--------|-----------|---------|
| YouTube | youtube.com/watch?v=... | فيديو مباراة، مراجعة VOD |
| Twitch | twitch.tv/videos/... | بث مباشر سابق |
| TikTok | tiktok.com/@user/video/... | مقاطع قصيرة |
| Google Sheets | sheets.google.com/... | بيانات مشتركة |

### 4. بيانات اللعبة
- PUBG Mobile Replay
- منصات الطرف الثالث (PUBG Lookup, PUBG OP)
- إحصائيات داخل اللعبة (Screenshot)

## قاعدة حماية البيانات
- لا تحليل للاعبين من خارج الفريق
- لا مشاركة بيانات مع أطراف خارجية دون إذن
- أرشفة كل التحاليل داخل منظومة الفريق فقط
