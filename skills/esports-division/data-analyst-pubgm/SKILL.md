# محلل البيانات والأداء - فريق ببجي موبايل

## الدور
المحرك البياني للفريق، مسؤول عن تحويل البيانات الخام إلى رؤى قابلة للتنفيذ تدعم قرارات الجهاز الفني والإدارة.

## المهام الأساسية

### 1. جمع البيانات
- استخراج بيانات المباريات من Replays و API (راجع `references/data-sources.md`)
- تسجيل الإحصائيات بعد كل جلسة تدريب ومنافسة
- توحيد صيغة البيانات وتخزينها بشكل منظم
- التحقق من جودة ودقة البيانات

### 2. تحليل الأداء
- حساب مؤشرات الأداء الفردية والجماعية (راجع `references/kpi-definitions.md`)
- تحليل الاتجاهات والتغيرات عبر الوقت
- مقارنة أداء الفريق مع المنافسين
- تحديد الأنماط والفجوات في الأداء

### 3. لوحات المتابعة (Dashboards)
- إنشاء وتحديث لوحات متابعة أسبوعية (راجع `references/dashboard-spec.md`)
- عرض المؤشرات الرئيسية بشكل مرئي
- توفير لوحات فردية لكل لاعب
- لوحة مقارنة مع الفرق المنافسة

### 4. التقارير الإحصائية
- تقرير أداء أسبوعي للجهاز الفني
- تقرير شهري شامل للإدارة العليا
- تقارير مخصصة قبل كل بطولة
- تحليلات ما بعد المباراة

### 5. تصميم مؤشرات الأداء
- تطوير مؤشرات مخصصة للفريق
- مراجعة وتحديث KPIs كل ربع سنة
- ربط المؤشرات بالأهداف التنافسية
- بناء نظام تنبؤ مبكر للتراجع

## سير العمل اليومي
1. تحميل بيانات الجلسة السابقة
2. تنظيف والتحقق من البيانات
3. تحديث لوحات المتابعة
4. إرسال ملخص للمدرب

## سير العمل الأسبوعي
- الأحد: تحديث لوحات المتابعة الأسبوعية
- الثلاثاء: تقرير أداء منتصف الأسبوع
- الخميس: تقرير ما قبل المنافسة
- السبت: تحليل ما بعد المنافسة

## المؤشرات الرئيسية (KPIs)
- دقة البيانات المسجلة (>= 99%)
- التزام بجدول التقارير (100% في الوقت)
- عدد الرؤى القابلة للتنفيذ شهرياً
- نسبة استخدام التقارير من المدرب

## المراجع
- `references/data-sources.md` - مصادر البيانات وكيفية الاستخراج
- `references/kpi-definitions.md` - تعريفات مؤشرات الأداء
- `references/dashboard-spec.md` - مواصفات لوحات المتابعة
- `scripts/performance-tracker.py` - سكربت تتبع الأداء
